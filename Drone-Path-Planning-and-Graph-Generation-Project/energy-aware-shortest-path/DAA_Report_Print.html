Energy-Aware Shortest Path Algorithm with Environmental Factors

1. Problem Definition

Classical shortest path routing minimizes distance, but real navigation may require minimizing energy. This project compares Standard Dijkstra with a Modified Dijkstra that includes environmental factors.

cost = distance + wind_effect + altitude_cost

2. Objective

  1. Implement baseline Standard Dijkstra.
  2. Implement Modified Dijkstra with environmental cost.
  3. Add obstacle-aware routing (no-fly zones).
  4. Add sudden wind-shift support (time-dependent costs).
  5. Compare outputs visually and numerically.

3. Environment Model

  • 2D grid graph; each cell is a node.
  • 4-neighbor movement (up, down, left, right).
  • Wind direction and strength influence directional movement cost.
  • Altitude values per cell model climb and descent energy.
  • Blocked cells are treated as obstacles and removed from traversal.

3.1 Wind Effect

wind_effect = wind_strength * (1 - dot(move_unit, wind_unit))

3.2 Altitude Cost

climb = max(0, altitude_to - altitude_from), descent = max(0, altitude_from - altitude_to)
altitude_cost = altitude_factor * (climb + downhill_factor * descent)

3.3 Dynamic Wind Shift

if step < shift_step use (direction_before, strength_before), else use (direction_after, strength_after)

4. Algorithm Pseudocode

4.1 Standard Dijkstra

DIJKSTRA_DISTANCE(grid, start, goal):
  initialize dist[v] = INF, parent[v] = NIL
  dist[start] = 0
  push (0, start) into min-priority queue

  while queue not empty:
    (d, u) = pop_min()
    if d > dist[u]: continue
    if u == goal: break

    for each neighbor v of u:
      w = geometric_distance(u, v)
      if dist[u] + w < dist[v]:
        dist[v] = dist[u] + w
        parent[v] = u
        push (dist[v], v)

  return path, dist[goal]

4.2 Modified Dijkstra

DIJKSTRA_ENERGY(grid, altitude, start, goal, wind_direction, wind_strength, altitude_factor):
  initialize dist[v] = INF, parent[v] = NIL
  dist[start] = 0
  push (0, start) into min-priority queue

  while queue not empty:
    (d, u) = pop_min()
    if d > dist[u]: continue
    if u == goal: break

    for each neighbor v of u:
      distance = geometric_distance(u, v)
      wind_effect = wind_strength * (1 - dot(move_unit(u, v), wind_unit(wind_direction)))
      climb = max(0, altitude[v] - altitude[u])
      descent = max(0, altitude[u] - altitude[v])
      altitude_cost = altitude_factor * (climb + downhill_factor * descent)
      w = distance + wind_effect + altitude_cost

      if dist[u] + w < dist[v]:
        dist[v] = dist[u] + w
        parent[v] = u
        push (dist[v], v)

  return path, dist[goal]

4.3 Time-Expanded State (Dynamic Wind Mode)

State is extended from node to (node, step).
Edge cost is evaluated using wind profile active at that step.
This supports sudden wind changes during traversal while preserving Dijkstra logic
under non-negative weights.

5. Complexity Analysis

Let V be number of nodes and E be number of edges. For a 4-neighbor grid, E = O(V). Using a binary heap:

  • Time: O((V + E) log V) = O(V log V)
  • Space: O(V)

In dynamic-wind mode, time-expanded search is used with horizon T: Time = O((T*V + T*E) log(T*V)), Space = O(T*V).

7. Conclusion

The modified Dijkstra keeps the same asymptotic efficiency as baseline Dijkstra but improves path realism by including environment-aware movement cost. In all tested scenarios, modified routing reduced energy cost while retaining feasible path lengths.
